## notes
* X軸Y軸で考える
* 到達したい座標がX, Yで与えられる
* 1回につき最大L距離分進める
* 進む回数と向きを変える回数が必要になる

e.g. (-7, 15)で1回7距離進めるロボットだと
左上に向かう必要がありますね。
最初は北を向いているので、どこかで一回左に90度転回が必要です
距離は横に7、縦に15必要なので、
7 / 7の切り上げで1
15 / 7 の切り上げで3 

合計移動命令が4回、転回命令が1回

* 移動の回数は縦横それぞれ移動可能距離の除算で切り上げすれば求められる
 * integerだと1/2は0.5にならないのでfloatで計算が必要です
 * ceilは数値が大きくなる方にまとめられるので今回は絶対を求めるabsのタイミングが重要
* 展開の数の判断
 * 正か負かで展開数が変わる
 * 時計周り or 反時計周りに90のどっちか
 * 最初は北を向いているのでYが正でもなにもしない
 * 0のときは不要
* ちょっと考慮が必要だった。
* Xが0以外なら左右のどちらかに転回する
* 基本、Yも負の時1度転回が必要（正は最初から北を向いているので不要）
* ただし、Xが0のときにYが負だと、向き変更なしから一気に反対を向くので2回転回が必要
 
e.g.
1, 4 

Xは正なので、90度時計回りへ
Yも正ですが、最初は北を向いているので何もしません
1回

-7, 15
Xは負なので、90度反時計回りへ
Yも正ですが、最初は北を向いているので何もしません
 1回
 
0, 2
Xは0なので転回しません
Yは正なので転回しません
