## notes
* 最大10^8枚のビスケットが食べたいはちょっと食べすぎ
* まず2のn乗でオーバしないところまで叩き続ける
 * オーバしたら差分を求めて、2で割った数を入れて叩くとできそう？
 * 10^8ということは2^nでいうと2^27までいくと超えちゃう。2^26が限界
 * timesは0からスタートするので28で
 
e.g. 
1枚の時は0回
2枚の時は1回
3枚の時は2回
4枚の時は2回
5枚の時は3回
6枚の時は、maxで2回叩いて、4枚。あと2枚欲しいので2枚残して、2枚を倍にすると、6枚ジャスト。つまり3回
7枚の時は、maxで2回叩いて、4枚。あと3枚ほしい。
1枚入れる場合、2+3で5枚
2枚入れる場合、4+2で6枚
3枚入れる場合、6+1で7枚！
そうか増やしたい数だけ入れるのか
8枚の時は2の3乗なので、3回。
9枚の時は、3回は確実。あと1枚増やしたいので、1枚入れると
7+2で9枚っと。

10枚の時も3回はいる。残り2枚欲しいので、6枚残して、2枚入れると6+4で10枚！！
こういう感じで、2のべき乗が繰り上がる段階で1回増える、と予測される。
つまり16までは4回でまかなえて、17からは32までは5回は必要。
たとえば31で考えると
2の4乗は必須なので、16枚確保される。残り15枚欲しいので15枚入れます。
残った一枚に30枚になったものを足すと31枚ですね〜！5回で足りるわけですよ。
100枚食べたい時。
2は6乗で64になって、7乗で128になります。なので7回叩けばちょうど100枚食べれるはずです
1000枚食べたい時は
2の8乗が256、9乗が512、10乗が1024なので、最低10回叩けばokです。

ここまでわかればあとは単純。2の27乗まででどこまでいったらオーバするか計算すれば良い。
